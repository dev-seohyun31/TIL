---

> I2C(Inter-Integrated Circuit)는 SCL(클럭)과 SDA(데이터) 두 선만으로  
> 여러 슬레이브를 연결할 수 있는 저속 직렬 통신이다.  
> 센서, EEPROM, OLED 디스플레이 등에 흔히 쓰인다.

---

## 1. iLLD 계층 구조 — I2C 관점

```
Your Code
    │
    ▼
[Hld]  IfxI2c_I2c.h / IfxI2c_I2c.c
       iLLD/TC3xx/Tricore/I2c/I2c/
       (마스터 init, 디바이스 등록, read/write 래핑)
    │
    ▼
[Lld]  IfxI2c.h / IfxI2c.c
       iLLD/TC3xx/Tricore/I2c/Std/
       (클럭 설정, FIFO 제어, 레지스터 접근)
    │
    ▼
[SFR]  I2C_CLC, FDIVCFG, ADDRCFG, TXFIFO, RXFIFO ...
    │
    ▼
[HW]   SCL / SDA 핀 (오픈 드레인, 풀업 저항 필요)
```

## 2. I2C 통신 기초 개념

```
I2C 버스
         풀업저항  풀업저항
           │        │
VCC ───────┤────────┤
           │        │
SCL ───────┴────────┘ (클럭, 마스터가 생성)
SDA ───────┬────────┐ (데이터, 양방향)
           │        │
        마스터    슬레이브(들)
```

**트랜잭션 구조:**
```
START │ ADDR(7bit) │ R/W │ ACK │ DATA │ ACK │ ... │ STOP
```

| 항목 | 설명 |
|---|---|
| START | SDA High→Low (SCL High 상태) |
| ADDR | 슬레이브 주소 7비트 (0x00~0x7F) |
| R/W | 0 = 쓰기, 1 = 읽기 |
| ACK | 슬레이브가 SDA를 Low로 당김 = 응답 |
| STOP | SDA Low→High (SCL High 상태) |

**주요 속도 모드:**

| 모드 | 속도 |
|---|---|
| Standard Mode | 100 kHz |
| Fast Mode | 400 kHz |
| Fast Mode Plus | 1 MHz |

## 3. I2C 모듈 동작 원리

```
쓰기 트랜잭션
마스터: START → [ADDR+W] → [REG_ADDR] → [DATA] → STOP
슬레이브:                ← ACK         ← ACK    ← ACK

읽기 트랜잭션 (레지스터 읽기 패턴)
마스터: START → [ADDR+W] → [REG_ADDR] → ReSTART → [ADDR+R] → ...    → STOP
슬레이브:                ← ACK        ← ACK                    [DATA] → NACK
```

읽기는 쓰기(레지스터 주소 지정)와 읽기(데이터 수신)를 **ReSTART로 연결**하는  
두 단계 트랜잭션인 경우가 대부분이다.

## 4. 핵심 레지스터 요약

| 레지스터 | 역할 |
|---|---|
| `I2C_CLC` | 모듈 클럭 enable |
| `I2C_FDIVCFG` | 클럭 분주 설정 (SCL 주파수) |
| `I2C_ADDRCFG` | 슬레이브 주소, R/W 방향 설정 |
| `I2C_MRPSCTRL` | 트랜잭션 바이트 수, ReSTART 설정 |
| `I2C_TXFIFO` | 송신 데이터 쓰기 |
| `I2C_RXFIFO` | 수신 데이터 읽기 |
| `I2C_RPSSTAT` | 트랜잭션 상태 (진행 중, 완료, 에러) |
| `I2C_PIRQSS` | 인터럽트 상태 플래그 |

## 5. iLLD API — IfxI2c_I2c

파일 위치: `iLLD/TC3xx/Tricore/I2c/I2c/IfxI2c_I2c.h`

| 함수 | 동작 |
|---|---|
| `IfxI2c_I2c_initConfig(config, i2c)` | 마스터 config 기본값 초기화 |
| `IfxI2c_I2c_initMaster(master, config)` | I2C 마스터 초기화 |
| `IfxI2c_I2c_initDeviceConfig(config, master)` | 디바이스(슬레이브) config 초기화 |
| `IfxI2c_I2c_initDevice(device, config)` | 슬레이브 디바이스 등록 |
| `IfxI2c_I2c_write(device, data, size)` | 슬레이브에 데이터 쓰기 |
| `IfxI2c_I2c_read(device, data, size)` | 슬레이브에서 데이터 읽기 |

I2C는 **마스터 → 디바이스** 2단계 구조다.  
슬레이브가 여러 개면 디바이스를 주소별로 따로 등록한다.

## 6. SPI vs I2C 핵심 차이

| 항목 | SPI (QSPI) | I2C |
|---|---|---|
| 신호선 수 | 4선 (SCLK/MOSI/MISO/CS) | 2선 (SCL/SDA) |
| 슬레이브 구분 | CS 핀 (슬레이브마다 1개) | 주소 (7비트) |
| 통신 방식 | 풀듀플렉스 | 반이중 |
| 속도 | 최대 수십 MHz | 최대 1 MHz |
| 배선 복잡도 | 슬레이브 많을수록 복잡 | 2선 고정 |
| 주 용도 | 플래시, DAC, 고속 센서 | EEPROM, 온도 센서, OLED |

## 7. 레지스터 방식 vs iLLD 비교

### 7-1. I2C 마스터 초기화

**레지스터 직접 접근**
```c
/* I2C 모듈 클럭 enable */
I2C0_CLC.B.DISR = 0u;

/* 400kHz Fast Mode: 100MHz / (분주값) = 400kHz */
I2C0_FDIVCFG.B.DEC = 0u;
I2C0_FDIVCFG.B.INC = 62u;   /* 계산값 */

/* 핀 라우팅 별도 설정 (오픈 드레인 모드) */
P13_IOCR0.B.PC0 = 0x18u;  /* SCL: open-drain output */
P13_IOCR0.B.PC1 = 0x18u;  /* SDA: open-drain output */
```

**iLLD**
```c
#include "IfxI2c_I2c.h"

IfxI2c_I2c        g_i2cMaster;
IfxI2c_I2c_Device g_i2cDevice;

IfxI2c_I2c_Config masterCfg;
IfxI2c_I2c_initConfig(&masterCfg, &MODULE_I2C0);

masterCfg.baudrate    = 400000u;  /* 400kHz */
masterCfg.pins.scl    = &IfxI2c0_SCL_P13_0_INOUT;
masterCfg.pins.sda    = &IfxI2c0_SDA_P13_1_INOUT;

IfxI2c_I2c_initMaster(&g_i2cMaster, &masterCfg);
```

오픈 드레인 핀 모드 설정과 클럭 분주비 계산이 내부에서 처리된다.

---

### 7-2. 데이터 송수신

**레지스터 직접 접근**
```c
/* 슬레이브 주소 + 쓰기 방향 설정 */
I2C0_ADDRCFG.B.ADR = 0x48u;  /* 슬레이브 주소 */
I2C0_ADDRCFG.B.RW  = 0u;     /* 쓰기 */
I2C0_MRPSCTRL.B.COUNT = 2u;  /* 2바이트 전송 */

/* TX FIFO에 데이터 쓰기 */
I2C0_TXFIFO.U = reg_addr;
I2C0_TXFIFO.U = data;

/* 트랜잭션 시작 → 완료 대기 */
I2C0_RUNCTRL.B.RUN = 1u;
while (I2C0_RPSSTAT.B.RPS != 0u);
```

**iLLD**
```c
uint8 buf[2] = {reg_addr, data};
IfxI2c_I2c_write(&g_i2cDevice, buf, 2u);
```

## 8. 예제 — 슬레이브 레지스터 1바이트 쓰기

> I2C0, 400kHz. 슬레이브 주소 0x48의 레지스터 0x01에 값 0xA5 쓰기.

```c
#include "IfxI2c_I2c.h"

static IfxI2c_I2c        g_i2cMaster;
static IfxI2c_I2c_Device g_i2cDevice;

static void I2C_init(uint8 slaveAddr)
{
    /* 마스터 초기화 */
    IfxI2c_I2c_Config masterCfg;
    IfxI2c_I2c_initConfig(&masterCfg, &MODULE_I2C0);
    masterCfg.baudrate = 400000u;
    masterCfg.pins.scl = &IfxI2c0_SCL_P13_0_INOUT;
    masterCfg.pins.sda = &IfxI2c0_SDA_P13_1_INOUT;
    IfxI2c_I2c_initMaster(&g_i2cMaster, &masterCfg);

    /* 슬레이브 디바이스 등록 */
    IfxI2c_I2c_deviceConfig devCfg;
    IfxI2c_I2c_initDeviceConfig(&devCfg, &g_i2cMaster);
    devCfg.deviceAddress = slaveAddr;
    IfxI2c_I2c_initDevice(&g_i2cDevice, &devCfg);
}

static void I2C_writeRegister(uint8 reg, uint8 val)
{
    /* [레지스터 주소, 데이터] 순서로 전송 */
    uint8 buf[2] = {reg, val};
    while (IfxI2c_I2c_write(&g_i2cDevice, buf, 2u) != IfxI2c_I2c_Status_ok);
}

int core0_main(void)
{
    I2C_init(0x48u);
    I2C_writeRegister(0x01u, 0xA5u);
    while (1) {}
    return 0;
}
```

## 9. 예제 — 슬레이브 레지스터 1바이트 읽기

읽기는 두 단계다.  
먼저 레지스터 주소를 쓴 뒤, 데이터를 읽는다.

```c
#include "IfxI2c_I2c.h"

extern IfxI2c_I2c_Device g_i2cDevice;  /* I2C_init()에서 초기화 */

static uint8 I2C_readRegister(uint8 reg)
{
    /* 1단계: 레지스터 주소 쓰기 (ReSTART 전 주소 지정) */
    while (IfxI2c_I2c_write(&g_i2cDevice, &reg, 1u) != IfxI2c_I2c_Status_ok);

    /* 2단계: 1바이트 읽기 */
    uint8 val = 0u;
    while (IfxI2c_I2c_read(&g_i2cDevice, &val, 1u) != IfxI2c_I2c_Status_ok);

    return val;
}

int core0_main(void)
{
    I2C_init(0x48u);

    uint8 val = I2C_readRegister(0x00u);
    /* val: 슬레이브 0x48의 레지스터 0x00 값 */

    while (1) {}
    return 0;
}
```

> `write` → `read` 사이에 iLLD가 내부적으로 **ReSTART** 조건을 생성한다.  
> STOP 없이 방향만 바꾸는 것이 I2C 레지스터 읽기의 표준 패턴이다.

## 10. 예제 — 여러 바이트 연속 읽기 (Burst Read)

> 슬레이브 0x48, 레지스터 0x00부터 6바이트 연속 읽기.  
> (예: MPU-6050 자이로/가속도 센서 원시 데이터 6바이트)

```c
#include "IfxI2c_I2c.h"

extern IfxI2c_I2c_Device g_i2cDevice;

#define BURST_LEN  6u

static void I2C_burstRead(uint8 startReg, uint8 *buf, uint8 len)
{
    /* 시작 레지스터 주소 지정 */
    while (IfxI2c_I2c_write(&g_i2cDevice, &startReg, 1u) != IfxI2c_I2c_Status_ok);

    /* 연속으로 len바이트 읽기 */
    while (IfxI2c_I2c_read(&g_i2cDevice, buf, len) != IfxI2c_I2c_Status_ok);
}

int core0_main(void)
{
    I2C_init(0x68u);  /* MPU-6050 기본 주소 */

    /* MPU-6050 power on (sleep 해제) */
    uint8 wake[2] = {0x6Bu, 0x00u};
    IfxI2c_I2c_write(&g_i2cDevice, wake, 2u);

    uint8 raw[BURST_LEN];
    while (1)
    {
        /* 가속도 XYZ 6바이트 (레지스터 0x3B~0x40) */
        I2C_burstRead(0x3Bu, raw, BURST_LEN);

        int16 accel_x = (int16)((raw[0] << 8u) | raw[1]);
        int16 accel_y = (int16)((raw[2] << 8u) | raw[3]);
        int16 accel_z = (int16)((raw[4] << 8u) | raw[5]);

        volatile uint32 i; for (i = 0; i < 100000u; i++);
    }
    return 0;
}
```

> I2C는 레지스터 주소를 보내면 슬레이브가 자동으로 주소를 증가시키며  
> 연속된 레지스터 값을 한 번에 보내주는 **Auto-Increment** 기능을 지원하는 경우가 많다.  
> 슬레이브 디바이스의 데이터시트에서 확인할 것.

## 정리

| 항목 | 레지스터 | iLLD |
|---|---|---|
| 마스터 초기화 | FDIVCFG 분주비 계산 + 핀 오픈드레인 설정 | `IfxI2c_I2c_initMaster()` |
| 슬레이브 등록 | ADDRCFG.ADR 직접 설정 | `IfxI2c_I2c_initDevice()` |
| 데이터 쓰기 | TXFIFO 직접 + RUNCTRL.RUN | `IfxI2c_I2c_write()` |
| 데이터 읽기 | 쓰기 후 방향 전환 + RXFIFO 읽기 | `IfxI2c_I2c_read()` |
| 완료 확인 | RPSSTAT.RPS 폴링 | 반환값 `IfxI2c_I2c_Status_ok` |

I2C는 SCL/SDA가 오픈 드레인이므로 **반드시 외부 풀업 저항**이 필요하다.  
400kHz에서는 보통 2.2kΩ~4.7kΩ, 100kHz에서는 4.7kΩ~10kΩ를 사용한다.  
풀업 저항 없이 연결하면 SDA가 HIGH로 올라가지 않아 ACK를 받지 못한다.
